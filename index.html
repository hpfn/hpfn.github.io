Python mixed in

Mixin

I was reading the Django code, middleware, and ends up with a
Python mixin. It was good to know that I did not understand the
subject correctly. I read about it but just read is not enough
when talking about programming.

I was unable to read the code because a class without inheritance
was calling 'super()'. I knew that you do not instantiate a mixin
directly. But did not know how to use it.

Henrique Bastos sent me an example that points out when the code
is mixed in the hierarchy:

class A:
.....def __init__(self):
.........print('A.__init__')

class Mixin:
....def __init__(self):
........print('Mixin.__init__')
........super().__init__()

class B(A, Mixin): pass

class C(Mixin, A): pass

.>>> A()
A.__init__

.>>> Mixin()
Mixin.__init__

.>>> B()
A.__init__

.>>> C()
Mixin.__init__
A.__init__

Mixins are used to extend a class. They provide optional customizations.
And Django project likes it.
