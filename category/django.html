Django tickets

First Ticket

2018-02-17

It was earlier than I expected, but it happened.

I found an "Easy picking" ticket with instructions
about what to do. It was open for months, which
means that it was not a big problem.

An "if" statement that raises a ValueError. That's
all. And some adjustments in the form test file.

The PR was accepted, and I enjoyed each moment.

One More Ticket

One more simple bug fixed.

I was studying the ticket at the same time another guy. He set
"owner" first. But it was not a problem at the end.

He created the Pull Request and sent a solution. Then I commented
submitting a patch that was simpler and trying to explain my point
of view. He kindly accepted my suggestion, made the change, and
the PR was merged.

What a way to start a year. Two simple bugs, that's true. But they
gave me a good feeling.

obs: my comments on the ticket page were kind of messy.
PR one thing, ticket another thing. Different places.

Security Features

Deployment Checklist

There is a big difference between development and production. The
settings file needs adjust when deploying a Django project.

There are many security features in Django, and some of them should
be enabled just before the deployment. A settings file for development
and a settings file for production.

The official documentation has a checklist. And a command that can
make things easier:

django-admin check --deploy
